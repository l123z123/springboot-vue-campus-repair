# 管理员报修流程规范化

> 状态：已批准 | 日期：2026-05-12

## 问题

管理员可以提交报修，但提交后：
1. 可以自己审批自己的工单（无校验）
2. 聊天界面被禁止发送消息（`canWrite` 限制 `role≤1`）

导致管理员报修流程不合逻辑。

## 方案

### 规则

| # | 规则 | 实现 |
|---|------|------|
| 1 | 管理员报修 → 跳过审核，初始状态直接为「待派单」 | 后端 `create()` 判断 role=2 时 status=3 而非 0 |
| 2 | 管理员可参与聊天 | 去掉 `canWrite` 的 role 限制 |
| 3 | 审核接口防自审 | `auditOrder` 判断审核人≠报修人，否则报错 |

### 流程对比

```
学生报修:   提交(0) → 审核(1→3) → 派单(4) → 维修(5) → 完成(6) → 确认(7) → 评价(8)
管理员报修: 提交 → 自动待派单(3) → 派单(4) → 维修(5) → 完成(6) → 确认(7) → 评价(8)
```

### 改动文件

| 文件 | 改动 |
|------|------|
| `RepairServiceImpl.java` — `create()` | role=2 时 status 设为 SUBMITTED→WAITING_DISPATCH(3)，跳过 WAITING_AUDIT |
| `RepairServiceImpl.java` — `auditOrder()` | 校验 reporterId ≠ 当前用户（管理员可审别人的单，不能审自己的） |
| `OrderChat.vue` — `canWrite` | `role === 0 || role === 1` → 去掉 role 限制 |
| `OrderDetail.vue` — 审核区域 | 管理员查看自己提交的工单时隐藏审核操作区 |

## 边界

- 管理员报修后仍需走派单→维修→完成→确认→评价完整流程，只是免审核
- 当前仅一个 admin 账号时，"另一个管理员审批"的场景不会出现
- 不影响学生和维修工现有流程

## 不在范围内

- 多人审批工作流
- 管理员角色细分
