# 校园报修管理系统 - 全面改造设计文档

**日期**: 2026-05-08
**目标**: 业务完整真实、企业级标准、适合毕业答辩

---

## 一、现状问题

1. **工单管理=个人中心**: AppLayout 侧栏"工单管理"指向 `/profile?scroll=records`，工单列表嵌在个人资料页
2. **工单详情布局反业务**: 左侧大图+基本信息，右侧核心内容区。图片占1/3屏幕，处理进度和操作被挤到右边
3. **维修工工作台臃肿**: 塞了太多内容，职责不清
4. **统计页花哨**: 仪表盘、雷达图等难以解释的图表
5. **预约维修是伪需求**: 校园后勤场景不存在预约制
6. **缺少独立工单管理页**: 学生端和维修工端都没有

**学生端（6项）**:

## 三、页面设计

### 3.1 OrderDetail.vue — 工单详情（重写）

自上而下单列卡片流：

1. **顶部状态栏**: 工单编号 + 状态标签 + 紧急度标签
2. **基本信息卡**: el-descriptions 三列横排（报修地点/报修人/报修时间/维修工/联系电话/故障分类）
3. **故障描述卡**: 文字描述 + 图片缩略图网格(120px)，点击放大预览
4. **处理进度卡**: el-steps 步骤条替代 timeline（提交→审核→派单→接单→维修→完成→确认→评价），已完成✅，进行中🔵，未开始⚪
5. **操作区卡**: 按角色+状态条件渲染（管理员审核派单/维修工接单完成/学生确认评价）
6. **评价详情卡**: 评分+评论（条件渲染）

筛选栏：[搜索框] [状态:全部/待处理/进行中/已完成/已关闭] [查询/重置]

```

操作按钮：接单 / 去维修 / 查看详情
```

### 3.5 Profile.vue — 个人中心（重写）

```
头像 + 姓名 + 学号/工号 + 部门 + 个性签名 + [编辑资料]
【账号安全】修改密码入口
【我的统计】报修总数 | 完成数 | 评价均分
```

### 3.6 Admin/Stats.vue — 数据统计（精简）

```
4个核心指标：总工单 | 今日新增 | 待处理 | 完成率
【报修趋势】折线图（近30天每日报修量）
【分类分布】柱状图（水电/门窗/网络/家具/其他）
【维修工绩效】el-table（排名/姓名/完成数/均分/平均耗时）
```

删除：仪表盘、雷达图。

### 3.7 SmartHome.vue — 首页（优化）

```
欢迎区：用户名 + 角色标签 + 日期
快捷操作卡片：发布报修 | 我的工单 | 消息中心 | 帮助反馈
【我的待办】最多3条待处理工单
【最近通知】最新3条
```

### 3.8 Publish.vue — 发布报修（优化）

```
报修地点：校区→区域→具体位置（级联或下拉联动）
故障分类：下拉（水电/门窗/网络/家具/墙面/电器/其他）
故障描述：textarea + 字数统计 + 占位提示
紧急程度：Radio（普通-不影响使用 / 紧急-影响正常使用 / 非常紧急-存在安全隐患）
上传图片：el-upload 多图，支持预览
联系电话：自动填充
[提交报修] [重置]
```

### 3.9 Message.vue — 消息中心（优化）

增加：未读/已读筛选、全选已读、类型图标优化、时间格式化

### 3.10 Notice.vue — 公告通知（优化）

增加：置顶标识、发布时间、发布人、分页

### 3.11 Help.vue — 帮助反馈（优化）

增加：FAQ 折叠面板（常见问题预设）+ 反馈表单

### 3.12 删除 Booking.vue

校园后勤维修不适用预约制，直接删除。
