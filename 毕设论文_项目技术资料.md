## 一、项目概述

### 1.1 项目名称
校园报修管理系统（Campus Repair Management System）

### 1.2 项目简介
基于 Vue3 + Spring Boot 的前后端分离校园报修管理系统（**派单版**业务与状态机以 `XQWD/` 与代码为准），面向学生、维修工、管理员三种角色，实现报修提交、**管理员审核派单、智能派单推荐**、工单流转、**邮箱验证码注册/登录（可配 QQ SMTP）**、WebSocket 实时通知、SLA 超时自动取消、数据统计、意见反馈、工单沟通等。

## 二、系统架构

### 2.1 总体架构
- **前后端分离**：Vue3 单页应用 + Spring Boot REST API（`context-path=/api`）
- **通信方式**：HTTP（Axios）+ **WebSocket（STOMP，原生 `ws` + `@stomp/stompjs`，无 SockJS）** 广播工单与通知
- **认证方式**：**JWT**；Token 可写入 **Redis** 以便登出失效（Redis 不可用时自动降级为仅校验 JWT，与现网实现一致）
- **注册/登录增强**：**邮箱验证码**（发信走 **Spring Mail + SMTP**，如 QQ 邮箱）；验证码默认存 **Redis**，**Redis 不可用时** 邮箱模块可 **内存降级**（单机验收可完成流程，多实例需 Redis）
- **消息通知**：WebSocket STOMP 直接推送，无需消息队列中间件
- **派单推荐**：管理员审核/派单时，系统基于维修工负载、同类经验、同区域经验和历史评分给出可解释推荐
- **定时任务**：**SLA 超时自动取消**（`SlaTimeoutMonitor`，每5分钟扫描，默认24小时）——超时未处理的工单（状态 0/1/3）自动标记为已取消，并通过 WebSocket 通知管理员

### 4.3 安全与权限
- **公开访问（示例）**：`/auth/login`、`/auth/login-email`、`/auth/register`、`/auth/send-code`、`/auth/send-email-code`、`/auth/reset-password`、`/files/**`、**`/ws/**`**、Knife4j/Swagger 相关路径；具体以 `SecurityConfig` 为准。
- **管理员接口**（Controller 中 `/admin/**` 等）：需 **`ROLE_ADMIN`**（数字角色 2 映射为 `ROLE_ADMIN` 的实现以 `JwtUtil`/安全配置为准）。
- **其余业务接口**：需 **Bearer JWT**；过滤器为 `JwtAuthenticationFilter`。

### 4.4 工单状态与流转（派单版）

主线示例：**0 待审核 → 1 审核中 → 3 待派单 → 4 已派单 → 5 维修中 → 6 维修完成 → 7 学生确认 → 8 已评价**；另有 **9 已拒绝、10 已取消** 等终态。合法转移以 **`RepairOrderStatus` + `RepairServiceImpl`** 为准。旧文档中「0 待接单、1 进行中、2 已完成、4 已评价」的简化表述已废弃。

### 4.5 实时通知、智能派单与 SLA 超时自动取消

- **WebSocket**：`NotifyServiceImpl` 经 `SimpMessagingTemplate` 向 **`/topic/notify`** 广播 JSON；`type` 包括 `CREATED`（新工单）、`STATUS_CHANGED`（状态变更）、`SLA_TIMEOUT`（超时自动取消）、`CHAT`（聊天新消息）。
- **智能派单推荐**：`RepairServiceImpl.recommendRepairmen` 四维度评分——活跃工单数（-20分/单，上限5单）、同类经验（+8分/单，上限5单）、同区域经验（+5分/单，上限5单）、历史评分（+平均分×4）；维修工并发上限3单。
- **SLA 超时自动取消**：`SlaTimeoutMonitor` 每5分钟执行（`@Scheduled(cron = "0 */5 * * * ?")`），扫描状态为 0/1/3 且超过 `repair.sla.pending-hours`（默认24小时）的工单，自动标记为已取消（状态10），记录备注"系统自动取消：超过N小时未处理"，并通过 WebSocket 推送通知管理员。

### 5.3 主要功能页面
- **响应式前端形态**：管理员端为 PC 后台；学生端/维修工端为 **移动优先响应式 Web**，小屏底部导航，PC 宽屏侧栏与卡片分栏，不称为“小程序”

## 八、论文可写技术要点

1. **前后端分离**：Vue3 + Vite + Spring Boot REST，职责清晰，便于维护。
2. **派单版工单状态机**：与校园后勤实际一致，采用管理员审核派单、维修工处理、学生确认评价的闭环状态机。
3. **JWT 与权限**：无状态 Token；可配合 Redis 做登出失效；Spring Security 区分公开接口与 `ROLE_ADMIN` 等。
4. **注册与真实身份验证**：**邮箱验证码**（Spring Mail + SMTP，QQ 邮箱授权码模式）、`sys_user.email` 唯一；支持 **邮箱验证码登录** 与 **密码登录** 并存。
5. **高并发与测试数据**：并发创建工单用例及 SQL 清理脚本、内存/Redis 策略（可简述）。
6. **WebSocket 实时推送**：STOMP over WebSocket 直推（无消息队列中间件），`/topic/notify` 广播工单状态变更与通知。
7. **SLA 超时自动取消**：定时任务按 `pending-hours` 扫描未闭环工单，WebSocket 推送携带 orderIds，管理端铃铛可点击跳转工单详情。
8. **Redis 缓存与降级**：Redis 不可用时 JWT 与邮箱验证码自动内存降级，保障系统可用性。
9. **智能派单推荐**：基于维修工负载、同类经验、区域经验和历史评价的可解释规则评分，辅助管理员派单。
10. **数据可视化**：ECharts 展示区域、状态、类型、时段、维修工绩效等。
11. **可观测与降级**：Redis 不可用时部分能力降级、全局异常 503/业务错误分层（可引用 `GlobalExceptionHandler` 设计）。
12. **容器与本地部署**：Docker Compose 启动 MySQL/Redis 及端口映射，与 `application.yml` 对齐。
