# 校园报修管理系统 - 最终答辩验收文档

## 一、答辩口径

- 项目名称：基于 SpringBoot + Vue 的校园报修管理系统。
- 前端形态：Vue3 + Vite 响应式 Web 应用；管理员端为 PC 后台，学生端和维修工端为移动优先响应式 Web，不按小程序交付。
- 核心业务：学生报修、管理员审核派单、维修工处理、学生确认评价、管理端统计与通知。
- 主要创新点：**基于多因素评分的智能派单推荐机制**。

## 二、创新点说明

本系统不依赖外部 AI API，而是采用本地可解释规则评分，保证答辩和本地部署稳定。

推荐因素包括：

- 维修工当前在办工单数量：任务越少分数越高。
- 同类故障处理经验：处理过相同故障类型则加分。
- 同区域处理经验：处理过相同校区/区域则加分。
- 历史评价分：评价越高略微加分。
- 紧急工单倾斜：紧急工单优先推荐空闲维修工。

管理员在工单详情页可看到推荐维修工、推荐分和推荐理由，并可点击“采用”填入派单对象。最终派单仍由管理员确认，避免黑盒决策。

## 三、推荐启动与导入顺序

在项目根目录执行：

```bash
docker-compose up -d mysql redis rabbitmq
```

初始化数据库：

```bash
mysql --default-character-set=utf8mb4 -h 127.0.0.1 -P 3307 -u root -p campus_repair < backend/src/main/resources/schema.sql
mysql --default-character-set=utf8mb4 -h 127.0.0.1 -P 3307 -u root -p campus_repair < backend/src/main/resources/init_data.sql
mysql --default-character-set=utf8mb4 -h 127.0.0.1 -P 3307 -u root -p campus_repair < init_all_accounts.sql
mysql --default-character-set=utf8mb4 -h 127.0.0.1 -P 3307 -u root -p campus_repair < sql/fix_user_display_data.sql
mysql --default-character-set=utf8mb4 -h 127.0.0.1 -P 3307 -u root -p campus_repair < sql/delete_concurrent_test_repair_orders.sql
mysql --default-character-set=utf8mb4 -h 127.0.0.1 -P 3307 -u root -p campus_repair < sql/seed_repair_orders_50.sql
```

旧库升级时先执行：

```bash
mysql --default-character-set=utf8mb4 -h 127.0.0.1 -P 3307 -u root -p campus_repair < sql/migration_add_sys_user_email.sql
```

启动后端与前端：

```bash
cd backend && mvn spring-boot:run
cd frontend && npm run dev
```

## 五、主流程验收

### 1. 学生注册与登录

- 打开 `/register`。
- 输入用户名、邮箱、验证码、密码、姓名、手机号、院系。
- 验证码通过 QQ 邮箱接收；若 Redis 不可用，后端可使用内存降级。
- 注册成功后可通过密码登录或邮箱验证码登录。

### 2. 学生提交报修

- 学生进入首页，点击发布报修。
- 填写校区、区域、类型、描述、联系电话，可上传图片。
- 提交后状态为 `0 待审核`。

### 3. 管理员审核与智能派单

- 管理员进入工单详情。
- 查看“智能派单推荐”区域。
- 确认推荐理由：在办数量、同类经验、同区域经验、评分。
- 可点击“采用”后提交审核并派单，也可以先进入待派单再派单。
- 派单后状态为 `4 已派单`，维修工字段写入。

### 4. 维修工处理

- 维修工登录后只看到管理员派给自己的工单。
- 点击工单详情，执行“开始维修”，状态变为 `5 维修中`。
- 完成维修后状态变为 `6 维修完成`。

### 5. 学生确认与评价

- 学生查看该工单，确认维修完成。
- 状态变为 `7 学生确认`。
- 提交评分与评价，状态变为 `8 已评价`。

### 6. 通知与 SLA 预警

- 新工单、状态变更会通过 WebSocket 推送。
- 超过 SLA 阈值的未闭环工单会触发 `SLA_TIMEOUT` 管理员提醒。
- 管理端铃铛可查看涉及工单号并跳转详情。

## 七、答辩风险排查

- 不要称学生端/维修工端为小程序，应称为“Vue3 响应式 Web / H5”。
- 不要说系统采用外部 AI 接口；本项目采用本地可解释规则评分，稳定、可控、可答辩。
- 不要再介绍早期尝试过的自由接单模式；最终业务口径是管理员统一派单。
- 若邮件发不出去，检查 `application.yml` 中 `spring.mail.username/password`，password 应为 QQ 邮箱授权码。
- 若 1045，检查 MySQL root 密码是否与 `docker-compose.yml` 一致。
- 若旧库注册失败，先执行 `sql/migration_add_sys_user_email.sql`。
